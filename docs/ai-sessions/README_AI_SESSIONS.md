# Purpose

Esta carpeta contiene el registro histórico de todas las sesiones de trabajo realizadas con inteligencia artificial durante el desarrollo de **Narrative Engineering**.

Cada sesión constituye un documento independiente que resume una conversación de trabajo y preserva el contexto en el que evolucionaron las ideas del proyecto.

Este documento responde únicamente a la siguiente pregunta:

> **¿Qué contiene esta carpeta y cómo deben registrarse las sesiones?**

---

# Scope

Las sesiones documentan el proceso de trabajo.

No documentan:

- metodología del proyecto
- decisiones oficiales
- canon del universo
- cambios del repositorio

Su objetivo es conservar la memoria del proceso creativo.

---

# Why One File Per Session?

Cada sesión representa una unidad independiente de conocimiento.

Registrar una sesión por archivo ofrece las siguientes ventajas:

- facilita la navegación del repositorio
- mejora la trazabilidad mediante Git
- evita archivos excesivamente grandes
- permite consultar sesiones individuales sin depender de un único documento
- mantiene una estructura escalable a largo plazo

---

# File Naming Convention

Cada sesión utilizará el siguiente formato.

```text
YYYY-MM-DD-session-XXX.md
```

Ejemplos.

```text
2026-07-20-session-001.md

2026-07-22-session-002.md

2026-08-03-session-003.md
```

El orden cronológico queda determinado automáticamente por el nombre del archivo.

---

# Session Template

Cada sesión deberá utilizar la siguiente estructura.

```markdown
# Session XXX

## Date

## Author

## AI

## Model

## Objective

## Topics

## Summary

## Outcomes

## Key Decisions

## Documents

## Next Steps

## Notes
```

Cada sesión deberá ser breve, objetiva y fácil de consultar.

No es necesario registrar la conversación completa.

---

# Documentation Principles

Toda sesión deberá cumplir las siguientes reglas.

- Documentar únicamente la información relevante.
- Resumir las conversaciones, no transcribirlas.
- Mantener objetividad.
- Registrar únicamente los resultados obtenidos.
- Identificar claramente los próximos pasos cuando existan.

---

# Relationship with Other Documents

Las sesiones representan la memoria del proceso de trabajo.

No sustituyen la documentación oficial.

Si durante una sesión se toma una decisión permanente, esta deberá documentarse en el archivo correspondiente según la arquitectura documental del proyecto.

Las sesiones únicamente registran que dicha decisión ocurrió.

---

# Final Statement

Las conversaciones también forman parte de la historia del proyecto.

Documentarlas permite comprender cómo evolucionaron las ideas antes de convertirse en documentación oficial.

Cada sesión representa un paso en la construcción del universo.

---

> *"Ideas become knowledge when they are documented. Sessions preserve that transformation."*
